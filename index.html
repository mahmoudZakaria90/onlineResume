Responsibilities

Well, according to:

  • The data gathering meeting which is between the client, GM, operation manager and account manager.

My Responsibilities are:

  • Understand the project documentation well, what is the product/service? its purposes for the business and the benefits which will return for the end user, for what type of end user will target?, what kind of it? if it's a website (informational) or a web/mobile application (System), its functionality, how it works and what techs/tools will be taken to do it?.
  • Set small meetings with the ui/ux designer during the process of project design step to test the usability of the website and to make sure that we are covering good user experience and meets the user needs.
  • After the client's approval I get my front-end workflow ready to start coding the delivered project design file (.PSD/.AI/.PDF) which include front and internal pages in order to transform it to be visual and interactive components on the web browser for the user who is going to interact with, with taking care of SEO principles.

  • Getting my workflow ready depends on important steps:

    • Going in depth with the design file I don’t just deliver it and say “Hey, let’s get this over !”, I extract first what is must be extracted like: logo, colors, typography(family, sizes, weights), images/background sizes and ratios, patterns and the reset of visual cues.
    • Declaring the repeated components or potential repeated/editable components in project's designs files, so I Always keep in mind writing reusable and editable code (Components).
    • Make my workflow serve me in creating web pages faster, abstracted code, less time and efforts and of course with high quality of code.
  • Test the website on the most web browsers outputs (cross-browser compatibility).
  • Make the website respond to all screens and most used devices: “HD/MD screens, tablets, mobile phones (Responsive design) with fixing the differences between devices (cross-devices compatibility).
  • Deliver to the ui/ux designer for more testing and matching between the design components and browser components.
  • After that, I deliver a readable/parsed HTML design to the back-end developer to start creating the dynamic pages .
